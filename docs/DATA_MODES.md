# 数据来源模式

`DATA_MODE` 是机器人遥测数据的总开关，默认值为 `debug`。摄像头/视频输入和 YOLO 检测结果仍按各自配置运行，不由这个开关伪造。

| 模式 | 统一电控遥测 | 旧 OPGPS | 缺失数据 | 虚拟果树事件 |
| --- | --- | --- | --- | --- |
| `real` | 按配置读取，逐字段使用真实值 | 仅作 GPS 兼容回退 | 保持为空，不回退 | 禁止 |
| `debug` | 按配置读取，逐字段优先使用 | 可作 GPS 兼容回退 | 对应字段用虚拟值补齐 | 允许 |
| `simulation` | 不打开 | 不打开 | 全部使用虚拟值 | 允许 |

## 配置开关

默认模式在 `config/app_config.py`：

```python
DATA_MODE = "debug"
```

也可以在启动时覆盖：

```bash
python main.py --data-mode real
python main.py --data-mode debug
python main.py --data-mode simulation
```

Linux 部署脚本支持环境变量，未指定时为 `debug`：

```bash
DATA_MODE=real ./detect.sh
```

甲方 B 的本地演示脚本已显式指定 `simulation`，不会读取或混入现场电控串口数据。

## 真实数据优先级

1. 58 字节统一电控遥测中的有效字段。
2. 统一遥测没有速度有效位时，根据连续有效 GPS 坐标估算速度。
3. 统一遥测没有有效 GPS 时，可以回退到单独的旧 OPGPS 串口。
4. 只有 `debug` 模式允许继续用虚拟传感器补齐仍缺失的字段。

GPS 估速会使用最小时间间隔、最小位移、速度上限和指数平滑过滤。只要电控速度有效位存在，系统始终优先使用电控速度，不使用估算值。

## 缺失值处理

真实模式不会再生成固定北京坐标、24V 电压、85% 电量或零速度等占位数据：

- 界面显示 `—`。
- HTTP JSON 使用 `null` 表示字段不存在，检测结果和图像仍可上报。
- UDP 是固定长度二进制协议，无法表示空值；必需遥测缺失时跳过该数据包，并每 5 秒最多记录一次缺失字段告警。

统一电控遥测可以提供机器人状态、路线、路径点、当前及左右果树编号、GPS、方向、速度、相机高度、电池和故障码。字段有效性完全由协议中的 `valid_flags` 决定；有效值为 0 仍然是真实数据。

真实果树数据有效但编号为 0 时，表示当前无树，不会再根据病害识别或演示逻辑生成虚拟树号。
